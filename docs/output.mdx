---
title: Output format
description: What Wellvector writes to disk and how to read the results.
---

Each run writes to a new folder at `Runs/run{N}/`. Run folders are never overwritten.

## Files

| File | Description |
|---|---|
| `casing_data.csv` | Clean output — 8 core columns |
| `casing_data_full.csv` | Same rows plus provenance and QA columns |
| `casing_conflicts.csv` | Per-field conflict log |
| `run_info.json` | Config snapshot, filter, timestamp, and cost breakdown |
| `fragments.json` | Raw collected fragments (used by `--resynth`) |
| `doc_scan_results.json` | Phase 0 scan output per document |

PDFs are cached under `pdfs/` and shared across all runs. The fragment cache at `pdfs/fragment_cache.json` is also shared.

## casing_data.csv columns

| Column | Description |
|---|---|
| `wellbore` | Wellbore name (e.g. `7/11-3`) |
| `casing_type` | `conductor`, `surface`, `intermediate`, `production`, `liner`, `LOT`, `FIT`, or `DST` |
| `casing_diameter_in` | Casing outer diameter in inches (e.g. `13.375` for 13 3/8") |
| `casing_depth_m` | Casing shoe depth in metres RKB |
| `hole_diameter_in` | Drilled hole diameter in inches |
| `hole_depth_m` | Hole depth in metres RKB |
| `mud_weight_g_cm3` | Mud weight at time of LOT/FIT or DST, in g/cm³ |
| `formation_test_type` | `LOT`, `FIT`, or `DST` where applicable |

<Note>
  Diameters are written as standard fractional strings in the full output (e.g. `13 3/8"`, `9 5/8"`). Depths use a comma as the decimal separator (e.g. `3492,8`) for compatibility with European spreadsheet software.
</Note>

## casing_conflicts.csv

When multiple sources provide different values for the same field, both values are logged here along with the resolution:

| Column | Description |
|---|---|
| `wellbore` | Wellbore name |
| `casing_type` | Which casing string was affected |
| `field` | Which field had a conflict |
| `chosen_value` | The value that was kept |
| `rejected_value` | The value that was discarded |
| `chosen_source` | Document the chosen value came from |
| `rejected_source` | Document the rejected value came from |
| `reason` | Plain-English explanation of the resolution |

### Example conflict entry

```
wellbore:        7/11-3
casing_type:     conductor
field:           casing_depth_m
chosen_value:    172
rejected_value:  176
chosen_source:   159_7_11_3_WELL_COMPLETION_REPORT-1 (REPORTED BY LICENSEE, priority 2)
rejected_source: 159_01_WDSS_General_Information (OLD NPD WDSS, priority 1)
reason:          Both Completion Reports agree on 172 m RKB. WDSS value of 176 m
                 discarded due to lower document priority.
```

## run_info.json

A snapshot of the configuration used for the run, plus cost:

```json
{
  "run_dir": "Runs/run13",
  "timestamp": "2026-03-26T08:42:05.036640+00:00",
  "script": "extract_casing_data.py",
  "config": {
    "TARGET_DOC_TYPES": ["REPORTED BY LICENSEE", "OLD NPD WDSS"],
    "MAX_CONCURRENT_PDFS": 1,
    "MAX_CONCURRENT_CLAUDE": 2,
    "SYNTH_TOKEN_LIMIT": 140000
  },
  "filter": { "wellbore": "7/11-3" },
  "cost": {
    "total_usd": 0.1718,
    "breakdown": {
      "claude-sonnet-4-6": {
        "input_tokens": 29138,
        "output_tokens": 5627,
        "cost_usd": 0.1718
      }
    }
  }
}
```
